H610CおよびH615Cノードを交換してください

2020年10月20日寄稿者 このページのPDFをダウンロード

CPU、マザーボードに関連する計算ノードの障害を修復するため、または電源が入らない場合は、シャーシを交換する必要があります。 NetApp HCI Bootstrap OSバージョン1.6以降を実行するH610C計算ノードに障害のあるDIMMがある場合は、DIMMを交換でき、シャーシを交換する必要はありません。 H615Cノードの場合、DIMMに障害が発生した場合にシャーシを交換する必要はありません。故障したDIMMのみを交換できます。

H610CとH615Cの場合、ノードとシャーシは別個のコンポーネントではないため、「ノード」と「シャーシ」という用語は同じ意味で使用されます。
必要なもの

  • ノードに障害が発生したことを確認しました。

  • 交換用のシャーシがあります。
    交換品を注文するには、ネットアップのサポートにお問い合わせください。

  • 静電放電（ESD）リストバンドを使用しているか、他の帯電防止保護を行っています。

  • シャーシに接続されている各ケーブルにラベルを付けました。

このタスクについて

VMware vSphere Web Clientのアラームは、ホストに障害が発生したときに警告を発します。 VMware vSphere Web Clientで障害が発生したホストのシリアル番号を、ノードの背面にあるステッカーのシリアル番号と一致させる必要があります。

手順の概要

この手順の手順の概要は次のとおりです。
ノードを交換する準備をする
ノードを交換してください
GPUドライバーをインストールします
ノードをクラスターに追加します

ノードを交換する準備をする

ノードを交換する前に、ノードでホストされている仮想マシン（VM）を使用可能なホストに移行し、クラスターからノードを削除する必要があります。シリアル番号やネットワーク情報など、ノードに関する詳細を取得する必要があります。

デュアルインラインメモリモジュール（DIMM）の障害など、ノードがまだオンラインで機能しているコンポーネント障害の場合は、障害が発生したノードを削除する前に、クラスターからドライブを削除する必要があります。
ステップ

  1. VMware vSphere Web Clientで、VMを別の使用可能なホストに移行する手順を実行します。

    移行手順については、VMwareのドキュメントを参照してください。

  2. 障害が発生したノードを選択し、[モニター]> [ハードウェアステータス]> [センサー]を選択します。

  3. 障害が発生したノードのシリアル番号をメモします。次のスクリーンショットは単なる例です。

    記載した番号をノードの背面にあるステッカーのシリアル番号と照合してシャーシを識別するには、シリアル番号が必要です。

  4. 障害が発生したノードを右クリックし、「接続」>「切断」を選択します。

  5. [はい]を選択してアクションを確認します。

  6. 失敗したノードを右クリックし、[インベントリから削除]を選択します。

  7. [はい]をクリックしてアクションを確認します。

ノードを交換してください

障害が発生したノードをクラスターから削除した後、障害が発生したシャーシを削除して、交換用のシャーシを取り付けることができます。

ここでの手順を実行する前に、帯電防止保護が施されていることを確認してください。
ステップ

  1. 新しいシャーシを開梱し、平らな面に置きます。
    故障したシャーシをネットアップに返却するときのために、梱包材を保管してください。

  2. 取り外すシャーシの背面に挿入されている各ケーブルにラベルを付けます。
    新しいシャーシを取り付けたら、ケーブルを元のポートに戻す必要があります。

  3. シャーシの背面からすべてのケーブルを外します。

  4. 取り付け耳のつまみネジを緩めて、シャーシを取り外します。
    故障したシャーシをパッケージ化してNetAppに返送する必要があります。

  5. H615Cのみ。 DIMMをシャーシから取り外します。これらのDIMMを交換用シャーシに挿入します。

  6. シャーシの両側にある2つの電源ユニットを取り外します。
    これらの電源ユニットは、新しいシャーシで使用できます。

  7. オプション：レールを取り外し、交換用シャーシに付属の新しいレールを取り付けます。
    既存のレールを再利用する場合は、この手順をスキップできます。

  8. 交換用シャーシをレールにスライドさせます。

    シャーシをレールにスライドさせるときは、過度の力を加えないようにしてください。

  9. 取り外したシャーシを平らな面に置きます。
    パッケージ化してネットアップに返送する必要があります。

  10. 電源ユニットを交換してください。

  11. H615Cのみ。故障したシャーシから取り外したDIMMを挿入し、交換用のシャーシに挿入します。

  12. 最初にケーブルを外したポートにケーブルを再接続します。
    ケーブルを外したときにケーブルに付けたラベルは、ガイドとして役立ちます。

    シャーシの背面にある通気孔がケーブルまたはラベルで塞がれていると、過熱によるコンポーネントの早期故障につながる可能性があります。
    ケーブルをポートに無理に押し込まないでください。ケーブル、ポート、またはその両方が損傷する可能性があります。

  13. シャーシの電源を入れます。

GPUドライバーをインストールします

H610CノードのようなNVIDIAグラフィックスプロセッシングユニット（GPU）を備えたコンピューティングノードでは、増加した処理能力を利用できるように、VMwareESXiにNVIDIAソフトウェアドライバーをインストールする必要があります。

ステップ

  1. ブラウザを開き、次のURLでNVIDIAライセンスポータルを参照します。
    https://nvid.nvidia.com/dashboard/

  2. 環境に応じて、次のいずれかのドライバーパッケージをコンピューターにダウンロードします。

    vSphereバージョン ドライバーパッケージ

    vSphere 6.0

    NVIDIA-GRID-vSphere-6.0-390.94-390.96-392.05.zip

    vSphere 6.5

    NVIDIA-GRID-vSphere-6.5-410.92-410.91-412.16.zip

    vSphere 6.7

    NVIDIA-GRID-vSphere-6.7-410.92-410.91-412.16.zip

  3. コンピューターでドライバーパッケージを抽出します。
    結果の.VIBファイルは、圧縮されていないドライバーファイルです。

  4. .VIBドライバーファイルをコンピューターから計算ノードで実行されているESXiにコピーします。次の各バージョンのコマンド例は、ドライバーが管理ホストの$ HOME / NVIDIA / ESX6.x /ディレクトリにあることを前提としています。 SCPユーティリティは、ほとんどのLinuxディストリビューションですぐに利用できます。または、Windowsのすべてのバージョンでダウンロード可能なユーティリティとして利用できます。

    オプション 説明

    ESXi 6.0

    scp $ HOME / NVIDIA / ESX6.0 / NVIDIA **。vibroot @ <ESXi_IP_ADDR>：/。

    ESXi 6.5

    scp $ HOME / NVIDIA / ESX6.5 / NVIDIA **。vibroot @ <ESXi_IP_ADDR>：/。

    ESXi 6.7

    scp $ HOME / NVIDIA / ESX6.7 / NVIDIA **。vibroot @ <ESXi_IP_ADDR>：/。

  5. 次の手順を使用して、ESXiホストにrootとしてログインし、NVIDIAvGPUマネージャーをESXiにインストールします。

    1. 次のコマンドを実行して、rootユーザーとしてESXiホストにログインします。
      ssh root@<ESXi_IP_ADDRESS>

    2. 次のコマンドを実行して、NVIDIAGPUドライバーが現在インストールされていないことを確認します。
      nvidia-smi
      このコマンドは、メッセージnvidia-smi: not foundを返す必要があります。

    3. 次のコマンドを実行して、ホストでメンテナンスモードを有効にし、VIBファイルからNVIDIAvGPUマネージャーをインストールします。
      esxcli system maintenanceMode set --enable true
      esxcli software vib install -v /NVIDIA**.vib
      「 Operation finished successfully 」というメッセージが表示されます。

    4. 次のコマンドを実行し、8つのGPUドライバーすべてがコマンド出力にリストされていることを確認します。
      nvidia-smi

    5. 次のコマンドを実行して、NVIDIAvGPUパッケージが正しくインストールおよびロードされたことを確認します。
      vmkload_mod -l | grep nvidia
      このコマンドは、次のような出力を返す必要がありますnvidia 816 13808

    6. 次のコマンドを実行してメンテナンスモードを終了し、ホストを再起動します。
      esxcli system maintenanceMode set –enable false
      reboot -f

  6. NVIDIA GPUを使用して新しくデプロイされた他の計算ノードについて、手順4〜6を繰り返します。

  7. NVIDIAドキュメントサイトの手順を使用して、次のタスクを実行します。

    1. NVIDIAライセンスサーバーをインストールします。

    2. NVIDIAvGPUソフトウェア用に仮想マシンゲストを構成します。

    3. 仮想デスクトップインフラストラクチャ（VDI）コンテキストでvGPU対応デスクトップを使用している場合は、NVIDIAvGPUソフトウェア用にVMwareHorizonViewを構成します。

ノードをクラスターに追加します

新しい計算ノードを使用するようにNetAppHCIを構成する必要があります。

必要なもの

  • 仮想分散スイッチを使用してノードをデプロイメントに追加する場合、NetAppHCIが使用しているvSphereインスタンスにはvSphereEnterprisePlusライセンスがあります。

  • NetAppHCIで使用されているvCenterまたはvSphereインスタンスのいずれもライセンスの有効期限が切れていません。

  • 既存のノードと同じネットワークセグメントに空きおよび未使用のIPv4アドレスがあります（新しいノードは、そのタイプの既存のノードと同じネットワークにインストールする必要があります）。

  • vCenter管理者アカウントの資格情報を用意しました。

ステップ

  1. Webブラウザーを開き、管理ノードのIPアドレスを参照します。例えば：
    https://<ManagementNodeIP&gt ;

  2. NetApp HCIストレージクラスター管理者の資格情報を入力して、NetApp Hybrid CloudControlにログインします。

  3. [インストールの展開]ペインで、[展開]を選択します。
    ブラウザがNetAppDeploymentEngineを開きます。

  4. NetApp HCIストレージクラスタ管理者の資格情報を入力して、NetApp DeploymentEngineにログインします。

  5. [ようこそ]ページで、[はい]を選択します。

  6. [エンドユーザーライセンス]ページで、次の操作を実行します。

    1. VMwareエンドユーザー使用許諾契約書をお読みください。

    2. 条件に同意する場合は、契約テキストの最後にある[同意する]を選択します。

  7. [続行]をクリックします。

  8. vCenterページで、次の手順を実行します。

    1. NetAppHCIインストールに関連付けられているvCenterインスタンスのFQDNまたはIPアドレスと管理者の資格情報を入力します。

    2. [続行]を選択します。

    3. 新しい計算ノードを追加する既存のvSphereデータセンターを選択するか、[新しいデータセンターの作成]を選択して新しい計算ノードを新しいデータセンターに追加します。

      [Create New Datacenter]を選択すると、[クラスター]フィールドに自動的に入力されます。

    4. 既存のデータセンターを選択した場合は、新しい計算ノードを関連付ける必要があるvSphereクラスターを選択します。

      NetApp HCIが、拡張用に選択したクラスターのネットワーク設定を認識できない場合は、管理、ストレージ、およびvMotionネットワークのvmkernelおよびvmnicマッピングがデプロイメントのデフォルトに設定されていることを確認してください。

    5. [続行]を選択します。

  9. [ESXi資格情報]ページで、追加する1つまたは複数の計算ノードのESXiルートパスワードを入力します。
    NetAppHCIの最初の展開時に作成されたものと同じパスワードを使用する必要があります。

  10. [続行]を選択します。

  11. 新しいvSphereデータセンタークラスターを作成した場合は、[ネットワークトポロジ]ページで、追加する新しい計算ノードに一致するネットワークトポロジを選択します。

    計算ノードが2ケーブルトポロジを使用していて、既存のNetAppHCI展開がVLANIDで構成されている場合にのみ、2ケーブルオプションを選択できます。

  12. [利用可能なインベントリ]ページで、既存のNetAppHCIインストールに追加するノードを選択します。

    一部の計算ノードでは、vCenterバージョンがサポートする最高レベルでEVCを有効にしてから、インストールに追加する必要がある場合があります。これらの計算ノードでEVCを有効にするには、vSphereクライアントを使用する必要があります。有効にした後、[インベントリ]ページを更新し、計算ノードの追加を再試行してください。

  13. [続行]を選択します。

  14. オプション：新しいvSphereデータセンタークラスターを作成した場合は、[ネットワーク設定]ページで、[既存のクラスターから設定をコピーする]チェックボックスをオンにして、既存のNetAppHCIデプロイメントからネットワーク情報をインポートします。
    これにより、各ネットワークのデフォルトゲートウェイとサブネット情報が入力されます。

  15. [ネットワーク設定]ページで、ネットワーク情報の一部が最初の展開から検出されました。新しい計算ノードはそれぞれシリアル番号でリストされているため、新しいネットワーク情報を割り当てる必要があります。新しい計算ノードごとに、次の手順を実行します。

    1. NetApp HCIがネーミングプレフィックスを検出した場合は、それを[Detected Naming Prefix]フィールドからコピーし、[Hostname]フィールドに追加する新しい一意のホスト名のプレフィックスとして挿入します。

    2. [管理IPアドレス]フィールドに、管理ネットワークサブネット内にある計算ノードの管理IPアドレスを入力します。

    3. [vMotion IPアドレス]フィールドに、vMotionネットワークサブネット内にある計算ノードのvMotionIPアドレスを入力します。

    4. [iSCSI A-IPアドレス]フィールドに、iSCSIネットワークサブネット内にある計算ノードの最初のiSCSIポートのIPアドレスを入力します。

    5. [iSCSI B-IPアドレス]フィールドに、iSCSIネットワークサブネット内にある計算ノードの2番目のiSCSIポートのIPアドレスを入力します。

  16. [続行]を選択します。

  17. [ネットワーク設定]セクションの[レビュー]ページで、新しいノードが太字で表示されます。セクションの情報を変更する必要がある場合は、次の手順を実行します。

    1. そのセクションの[編集]を選択します。

    2. 変更が完了したら、後続のページで[続行]を選択して、[レビュー]ページに戻ります。

  18. オプション：ネットアップがホストするSolidFire Active IQサーバーにクラスター統計とサポート情報を送信しない場合は、最後のチェックボックスをオフにします。
    これにより、NetAppHCIのリアルタイムのヘルスおよび診断モニタリングが無効になります。この機能を無効にすると、NetAppがNetApp HCIをプロアクティブにサポートおよび監視して、本番環境に影響が及ぶ前に問題を検出および解決する機能が削除されます。

  19. [ノードの追加]を選択します。
    NetApp HCIがリソースを追加および構成している間、進行状況を監視できます。

  20. オプション：新しい計算ノードがvCenterに表示されていることを確認します。
