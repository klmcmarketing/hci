NetApp Hybrid CloudControlを使用してボリュームを作成および管理する

2020年11月12日寄稿者 このページのPDFをダウンロード

ボリュームを作成し、そのボリュームを特定のアカウントに関連付けることができます。ボリュームをアカウントに関連付けると、iSCSIイニシエーターとCHAP資格情報を介してアカウントがボリュームにアクセスできるようになります。

作成時にボリュームのQoS設定を指定できます。

NetApp Hybrid Cloud Controlのボリュームは、次の方法で管理できます。

  • ボリュームを作成する

  • QoSポリシーをボリュームに適用します

  • ボリュームを編集する

  • ボリュームのクローン

  • ボリュームを削除します

  • 削除したボリュームを復元する

  • 削除したボリュームを削除します

ボリュームを作成する

NetApp Hybrid CloudControlを使用してストレージボリュームを作成できます。

ステップ

  1. NetApp HCIまたはElementストレージクラスターの管理者資格情報を入力して、NetApp Hybrid CloudControlにログインします。

  2. ダッシュボードから、左側のナビゲーションメニューでストレージクラスターの名前を展開します。

  3. [ボリューム]> [概要]タブを選択します。

  4. [ボリュームの作成]を選択します。

  5. 新しいボリュームの名前を入力します。

  6. ボリュームの合計サイズを入力します。

    デフォルトのボリュームサイズの選択はGB単位です。 GBまたはGiBで測定されたサイズを使用してボリュームを作成できます。
    1GB = 1 000 000000バイト
    1GiB = 1 073 741824バイト

  7. ボリュームのブロックサイズを選択します。

  8. [アカウント]リストから、ボリュームにアクセスする必要があるアカウントを選択します。

    アカウントが50を超える場合、リストは表示されません。入力を開始すると、オートコンプリート機能に選択可能な値が表示されます。

  9. サービス品質を設定するには、次のいずれかを実行します。

    1. 既存のQoSポリシーを選択します。

    2. [QoS設定]で、IOPSのカスタマイズされた最小値、最大値、およびバースト値を設定するか、デフォルトのQoS値を使用します。

      最大またはバーストIOPS値が20,000IOPSを超えるボリュームでは、単一のボリュームでこのレベルのIOPSを達成するために、高いキュー深度または複数のセッションが必要になる場合があります。

  10. [ボリュームの作成]をクリックします。

QoSポリシーをボリュームに適用します

NetApp Hybrid Cloud Controlを使用して、既存のストレージボリュームにQoSポリシーを適用できます。

ステップ

  1. NetApp HCIまたはElementストレージクラスターの管理者資格情報を入力して、NetApp Hybrid CloudControlにログインします。

  2. ダッシュボードから、左側のナビゲーションメニューでストレージクラスターの名前を展開します。

  3. [ボリューム] > [概要]を選択します。

  4. ボリュームテーブルの[アクション]列で、ボリュームのメニューを展開し、[編集]を選択します。

  5. 次のいずれかを実行して、サービス品質を変更します。

    1. 既存のポリシーを選択します。

    2. [カスタム設定]で、IOPSの最小値、最大値、バースト値を設定するか、デフォルト値を使用します。

      ボリュームでQoSポリシーを使用している場合は、カスタムQoSを設定して、ボリュームとのQoSポリシーの関連付けを削除できます。カスタムQoSは、ボリュームQoS設定のQoSポリシー値を上書きします。
      IOPS値を変更するときは、数十または数百単位で増分します。入力値には有効な整数が必要です。バースト値が非常に高いボリュームを構成します。これにより、システムは、ボリュームの持続的なIOPSを制限しながら、時折発生する大きなブロックの順次ワークロードをより迅速に処理できます。

  6. [保存]を選択します。

ボリュームを編集する

NetApp Hybrid Cloud Controlを使用すると、QoS値、ボリュームサイズ、バイト値の計算に使用される測定単位などのボリューム属性を編集できます。レプリケーションを使用するため、またはボリュームへのアクセスを制限するために、アカウントアクセスを変更することもできます。

このタスクについて

次の条件下で、クラスターに十分なスペースがある場合は、ボリュームのサイズを変更できます。

  • 通常の動作条件。

  • ボリュームエラーまたは障害が報告されています。

  • ボリュームのクローンが作成されています。

  • ボリュームが再同期されています。

ステップ

  1. NetApp HCIまたはElementストレージクラスターの管理者資格情報を入力して、NetApp Hybrid CloudControlにログインします。

  2. ダッシュボードから、左側のナビゲーションメニューでストレージクラスターの名前を展開します。

  3. [ボリューム] > [概要]を選択します。

  4. ボリュームテーブルの[アクション]列で、ボリュームのメニューを展開し、[編集]を選択します。

  5. 必要に応じて変更を加えます。

    1. ボリュームの合計サイズを変更します。

      ボリュームのサイズを増やすことはできますが、減らすことはできません。 1回のサイズ変更操作で1つのボリュームのみサイズ変更できます。ガベージコレクション操作とソフトウェアのアップグレードによって、サイズ変更操作が中断されることはありません。
      レプリケーションのボリュームサイズを調整する場合は、最初にレプリケーションターゲットとして割り当てられたボリュームのサイズを増やします。次に、ソースボリュームのサイズを変更できます。ターゲットボリュームのサイズはソースボリューム以上にすることができますが、小さくすることはできません。
      デフォルトのボリュームサイズの選択はGB単位です。 GBまたはGiBで測定されたサイズを使用してボリュームを作成できます。
      1GB = 1 000 000000バイト
      1GiB = 1 073 741824バイト

    2. 別のアカウントアクセスレベルを選択します。

      • 読み取り専用

      • 読み書き

      • ロック済み

      • レプリケーションターゲット

    3. ボリュームにアクセスする必要があるアカウントを選択します。

      入力を開始すると、オートコンプリート機能により、選択可能な値が表示されます。

    4. 次のいずれかを実行して、サービス品質を変更します。

      1. 既存のポリシーを選択します。

      2. [カスタム設定]で、IOPSの最小値、最大値、バースト値を設定するか、デフォルト値を使用します。

        ボリュームでQoSポリシーを使用している場合は、カスタムQoSを設定して、ボリュームとのQoSポリシーの関連付けを削除できます。カスタムQoSは、ボリュームQoS設定のQoSポリシー値を上書きします。
        IOPS値を変更するときは、数十または数百単位で増分する必要があります。入力値には有効な整数が必要です。バースト値が非常に高いボリュームを構成します。これにより、システムは、ボリュームの持続的なIOPSを制限しながら、時折発生する大きなブロックの順次ワークロードをより迅速に処理できます。

  6. [保存]を選択します。

ボリュームのクローン

単一のストレージボリュームのクローンを作成することも、ボリュームのグループのクローンを作成して、データのポイントインタイムコピーを作成することもできます。ボリュームのクローンを作成すると、システムはボリュームのスナップショットを作成してから、スナップショットによって参照されるデータのコピーを作成します。

あなたが始める前に

  • 少なくとも1つのクラスターを追加して実行する必要があります。

  • 少なくとも1つのボリュームが作成されました。

  • ユーザーアカウントが作成されました。

  • 使用可能なプロビジョニングされていないスペースは、ボリュームサイズ以上である必要があります。

このタスクについて

クラスターは、ボリュームごとに一度に最大2つの実行中のクローン要求と、一度に最大8つのアクティブなボリュームクローン操作をサポートします。これらの制限を超えるリクエストは、後の処理のためにキューに入れられます。

ボリュームのクローン作成は非同期プロセスであり、プロセスに必要な時間は、クローン作成するボリュームのサイズと現在のクラスターの負荷によって異なります。

クローンボリュームは、ソースボリュームからボリュームアクセスグループメンバーシップを継承しません。
ステップ

  1. NetApp HCIまたはElementストレージクラスターの管理者資格情報を入力して、NetApp Hybrid CloudControlにログインします。

  2. ダッシュボードから、左側のナビゲーションメニューでストレージクラスターの名前を展開します。

  3. [ボリューム]> [概要]タブを選択します。

  4. クローンを作成する各ボリュームを選択し、表示される[クローン]ボタンをクリックします。

  5. 次のいずれかを実行します。

    • 単一のボリュームのクローンを作成するには、次の手順を実行します。

      1. [ボリュームのクローン]ダイアログボックスで、ボリュームクローンのボリューム名を入力します。

        説明的な命名のベストプラクティスを使用します。これは、環境で複数のクラスターまたはvCenterServerが使用されている場合に特に重要です。

      2. アカウントのアクセスレベルを選択します。

        • 読み取り専用

        • 読み書き

        • ロック済み

        • レプリケーションターゲット

      3. ボリュームクローンのサイズをGBまたはGIBで選択します。

        クローンのボリュームサイズを大きくすると、ボリュームの最後に空き領域が追加された新しいボリュームになります。ボリュームの使用方法によっては、パーティションを拡張するか、空き領域に新しいパーティションを作成して使用する必要がある場合があります。

      4. ボリュームクローンに関連付けるアカウントを選択します。

      5. [クローンボリューム]をクリックします。

    • 複数のボリュームのクローンを作成するには、次の手順を実行します。

      1. [ボリュームのクローン]ダイアログボックスで、[新しいボリューム名のプレフィックス]フィールドにボリュームクローンのオプションのプレフィックスを入力します。

      2. ボリュームクローンの新しいタイプのアクセスを選択するか、アクティブなボリュームからアクセスタイプをコピーします。

      3. ボリュームクローンに関連付ける新しいアカウントを選択するか、アクティブなボリュームからアカウントの関連付けをコピーします。

      4. [クローンボリューム]をクリックします。

クローン作成操作を完了する時間は、ボリュームサイズと現在のクラスター負荷の影響を受けます。複製されたボリュームがボリュームリストに表示されない場合は、ページを更新してください。

ボリュームを削除します

Elementストレージクラスターから1つ以上のボリュームを削除できます。

このタスクについて

システムは、削除されたボリュームをすぐにパージしません。それらは約8時間利用可能です。 8時間後、それらはパージされ、使用できなくなります。システムがボリュームをパージする前にボリュームを復元すると、ボリュームはオンラインに戻り、iSCSI接続が復元されます。

スナップショットの作成に使用されたボリュームが削除されると、関連するスナップショットは非アクティブになります。削除されたソースボリュームが削除されると、関連する非アクティブなスナップショットもシステムから削除されます。

管理サービスに関連付けられている永続ボリュームが作成され、インストールまたはアップグレード中に新しいアカウントに割り当てられます。永続ボリュームを使用している場合は、ボリュームまたはそれに関連するアカウントを変更または削除しないでください。これらのボリュームを削除すると、管理ノードが使用できなくなる可能性があります。
ステップ

  1. NetApp HCIまたはElementストレージクラスターの管理者資格情報を入力して、NetApp Hybrid CloudControlにログインします。

  2. ダッシュボードから、左側のナビゲーションメニューでストレージクラスターの名前を展開します。

  3. [ボリューム] > [概要]を選択します。

  4. 削除するボリュームを1つ以上選択します。

  5. 次のいずれかを実行します。

    • 複数のボリュームを選択した場合は、表の上部にある[クイックフィルターの削除]をクリックします。

    • 単一のボリュームを選択した場合は、[ボリューム]テーブルの[アクション]列で、ボリュームのメニューを展開し、[削除]を選択します。

  6. [はい]を選択して削除を確認します。

削除したボリュームを復元する

ストレージボリュームが削除された後でも、削除後8時間以内に復元すれば、復元できます。

システムは、削除されたボリュームをすぐにパージしません。それらは約8時間利用可能です。 8時間後、それらはパージされ、使用できなくなります。システムがボリュームをパージする前にボリュームを復元すると、ボリュームはオンラインに戻り、iSCSI接続が復元されます。

ステップ

  1. NetApp HCIまたはElementストレージクラスターの管理者資格情報を入力して、NetApp Hybrid CloudControlにログインします。

  2. ダッシュボードから、左側のナビゲーションメニューでストレージクラスターの名前を展開します。

  3. [ボリューム] > [概要]を選択します。

  4. [削除済み]を選択します。

  5. [ボリューム]テーブルの[アクション]列で、ボリュームのメニューを展開し、[復元]を選択します。

  6. [はい]を選択してプロセスを確認します。

削除したボリュームを削除します

ストレージボリュームが削除された後、それらは約8時間使用可能のままになります。 8時間後、それらは自動的にパージされ、使用できなくなります。 8時間待ちたくない場合は、削除できます

ステップ

  1. NetApp HCIまたはElementストレージクラスターの管理者資格情報を入力して、NetApp Hybrid CloudControlにログインします。

  2. ダッシュボードから、左側のナビゲーションメニューでストレージクラスターの名前を展開します。

  3. [ボリューム] > [概要]を選択します。

  4. [削除済み]を選択します。

  5. パージする1つ以上のボリュームを選択します。

  6. 次のいずれかを実行します。

    • 複数のボリュームを選択した場合は、表の上部にある[クイックフィルターの削除]をクリックします。

    • 単一のボリュームを選択した場合は、[ボリューム]テーブルの[アクション]列で、ボリュームのメニューを展開し、[パージ]を選択します。

  7. [ボリューム]テーブルの[アクション]列で、ボリュームのメニューを展開し、[パージ]を選択します。

  8. [はい]を選択してプロセスを確認します。
