管理ノードをインストールします

2020年11月11日寄稿者 このページのPDFをダウンロード

構成に適したイメージを使用して、NetAppElementソフトウェアを実行しているクラスターの管理ノードを手動でインストールできます。

この手動プロセスは、管理ノードのインストールにNetApp DeploymentEngineを使用していないSolidFireオールフラッシュストレージ管理者およびNetAppHCI管理者を対象としています。

必要なもの

  • クラスタバージョンは、NetAppElementソフトウェア11.3以降を実行しています。

  • インストールでIPv4を使用します。管理ノード11.3はIPv6をサポートしていません。

    IPv6をサポートする必要がある場合は、管理ノード11.1を使用できます。

  • ネットアップサポートサイトからソフトウェアをダウンロードする権限があります。

  • プラットフォームに適した管理ノードイメージタイプを特定しました。

    プラットホーム インストールイメージタイプ

    Microsoft Hyper-V

    .iso

    KVM

    .iso

    VMware vSphere

    .iso、.ova

    Citrix XenServer

    .iso

    OpenStack

    .iso

  • （プロキシサーバーを使用する管理ノード12.0および12.2）プロキシサーバーを構成する前に、NetApp Hybrid CloudControlを管理サービスバージョン2.16に更新しました。

このタスクについて

Element12.2管理ノードはオプションのアップグレードです。既存の展開には必要ありません。

この手順を実行する前に、永続ボリュームと、それらを使用するかどうかを理解しておく必要があります。永続ボリュームはオプションですが、VMが失われた場合の管理ノード構成データの回復に推奨されます。

ステップ

  1. ISOまたはOVAをダウンロードし、VMを展開します

  2. 管理ノードadminを作成し、ネットワークを構成します

  3. 管理ノードを構成します

  4. コントローラアセットを構成する

  5. （NetApp HCIのみ）計算ノード資産を構成します

ISOまたはOVAをダウンロードし、VMを展開します

  1. ネットアップサポートサイトから、インストール用のOVAまたはISOをダウンロードします。

    要素ソフトウェア： https ： //mysupport.netapp.com/site/products/all/details/element-software/downloads-tab
    NetApp HCI： https ： //mysupport.netapp.com/site/products/all/details/netapp-hci/downloads-tab

    1. [最新リリースのダウンロード]をクリックして、EULAに同意します。

    2. ダウンロードする管理ノードイメージを選択します。

  2. OVAをダウンロードした場合は、次の手順に従ってください。

    1. OVAをデプロイします。

    2. ストレージクラスターが管理ノード（eth0）とは別のサブネット上にあり、永続ボリュームを使用する場合は、ストレージサブネット上のVM（eth1など）に2番目のネットワークインターフェイスコントローラー（NIC）を追加するか、管理ネットワークはストレージネットワークにルーティングできます。

  3. ISOをダウンロードした場合は、次の手順に従ってください。

    1. 次の構成で、ハイパーバイザーから新しい64ビット仮想マシンを作成します。

      • 6つの仮想CPU

      • 24GB RAM

      • 400GB仮想ディスク、シンプロビジョニング

      • インターネットアクセスとストレージMVIPへのアクセスを備えた1つの仮想ネットワークインターフェイス。

      • （SolidFireオールフラッシュストレージのオプション）ストレージクラスターへの管理ネットワークアクセスを備えた1つの仮想ネットワークインターフェイス。ストレージクラスターが管理ノード（eth0）とは別のサブネット上にあり、永続ボリュームを使用する場合は、2番目のネットワークインターフェイスコントローラー（NIC）をストレージサブネット（eth1）のVMに追加するか、管理ネットワークがストレージネットワークにルーティングします。

        この手順の後半で電源を入れるように指示する手順の前に、仮想マシンの電源を入れないでください。

    2. ISOを仮想マシンに接続し、.isoインストールイメージで起動します。

      イメージを使用して管理ノードをインストールすると、スプラッシュ画面が表示されるまでに30秒の遅延が発生する場合があります。

  4. インストールが完了したら、管理ノードの仮想マシンの電源を入れます。

管理ノードadminを作成し、ネットワークを構成します

  1. ターミナルユーザーインターフェイス（TUI）を使用して、管理ノードadminユーザーを作成します。

    メニューオプション間を移動するには、上矢印キーまたは下矢印キーを押します。ボタン間を移動するには、Tabキーを押します。ボタンからフィールドに移動するには、Tabキーを押します。フィールド間を移動するには、上矢印キーまたは下矢印キーを押します。

  2. 管理ノードネットワーク（eth0）を構成します。

    ストレージトラフィックを分離するために追加のNICが必要な場合は、別のNICを構成する手順を参照してください：ストレージネットワークインターフェイスコントローラー（NIC）を構成します。

管理ノードを構成します

  1. 管理ノードにSSHで接続します。

  2. SSHを使用して、次のコマンドを実行し、root権限を取得します。プロンプトが表示されたら、パスワードを入力します。

     sudo su

  3. NTPを使用して、管理ノードとストレージクラスター間で時刻が同期されていることを確認します。

    1. /etc/ntp.confファイルを編集します。

    2. デフォルトのタイムサーバーを、ストレージクラスターで使用されているものと同じNTPサーバーに変更します。

    3. vSphereで、VMオプションの[ゲスト時刻をホストと同期する]チェックボックスがオフになっていることを確認します。

      VMに将来変更を加える場合は、このオプションを有効にしないでください。

  4. 管理ノードのセットアップコマンドを構成して実行します。

    安全なプロンプトでパスワードを入力するように求められます。クラスタがプロキシサーバーの背後にある場合は、パブリックネットワークにアクセスできるようにプロキシ設定を構成する必要があります。 
     / sf / packages / mnode / setup-mnode --mnode_admin_user [username] --storage_mvip [mvip] --storage_username [username] --telemetry_active [true]

    1. 次の必須パラメーターのそれぞれについて、[]括弧（括弧を含む）の値を置き換えます。

      コマンド名の省略形は括弧（）内にあり、フルネームの代わりに使用できます。

      • --mnode_admin_user（-mu）[username] ：管理ノード管理者アカウントのユーザー名。これは、管理ノードへのログインに使用したユーザーアカウントのユーザー名である可能性があります。

      • --storage_mvip（-sm）[MVIPアドレス] ：Elementソフトウェアを実行しているストレージクラスターの管理仮想IPアドレス（MVIP）。

      • --storage_username（-su）[username] ：- --storage_mvipパラメーターで指定されたクラスターのストレージクラスター管理者ユーザー名。

      • --telemetry_active（-t）[true] ： ActiveIQによる分析のためのデータ収集を可能にするtrueの値を保持します。

    2. （オプション）：アクティブIQエンドポイントパラメーターをコマンドに追加します。

      • --remote_host（-rh）[AIQ_endpoint] ：アクティブIQテレメトリデータが処理のために送信されるエンドポイント。パラメータが含まれていない場合は、デフォルトのエンドポイントが使用されます。

    3. （推奨）：以下の永続ボリュームパラメーターを追加します。永続ボリューム機能用に作成されたアカウントとボリュームを変更または削除しないでください。変更すると、管理機能が失われます。

      • --use_persistent_volumes（-pv）[true / false、デフォルト：false] ：永続ボリュームを有効または無効にします。永続ボリューム機能を有効にするには、値trueを入力します。

      • --persistent_volumes_account（-pva）[account_name] ：- --use_persistent_volumesがtrueに設定されている場合は、このパラメーターを使用して、永続ボリュームに使用されるストレージアカウント名を入力します。

        永続ボリュームには、クラスター上の既存のアカウント名とは異なる一意のアカウント名を使用してください。永続ボリュームのアカウントを他の環境から分離しておくことが非常に重要です。

      • --persistent_volumes_mvip（-pvm）[mvip] ：永続ボリュームで使用されるElementソフトウェアを実行しているストレージクラスターの管理仮想IPアドレス（MVIP）を入力します。これは、複数のストレージクラスターが管理ノードによって管理されている場合にのみ必要です。複数のクラスターが管理されていない場合は、デフォルトのクラスターMVIPが使用されます。

    4. プロキシサーバーを構成します。

      • --use_proxy（-up）[true / false、デフォルト：false] ：プロキシの使用を有効または無効にします。このパラメーターは、プロキシサーバーを構成するために必要です。

      • --proxy_hostname_or_ip（-pi）[host] ：プロキシのホスト名またはIP。これは、プロキシを使用する場合に必要です。これを指定すると、 --proxy_portを入力するように求められます。

      • --proxy_username（-pu）[username] ：プロキシユーザー名。このパラメーターはオプションです。

      • --proxy_password（-pp）[password] ：プロキシパスワード。このパラメーターはオプションです。

      • --proxy_port（-pq）[ポート、デフォルト：0] ：プロキシポート。これを指定すると、プロキシホスト名またはIP（ --proxy_hostname_or_ip ）を入力するように求められます。

      • --proxy_ssh_port（-ps）[ポート、デフォルト：443] ：SSHプロキシポート。これはデフォルトでポート443になります。

    5. （オプション）各パラメーターに関する追加情報が必要な場合は、パラメーターヘルプを使用します。

      • --help（-h） ：各パラメーターに関する情報を返します。パラメータは、初期展開に基づいて必須またはオプションとして定義されます。アップグレードと再デプロイのパラメータ要件は異なる場合があります。

    6. setup-mnodeコマンドを実行しsetup-mnode 。

コントローラアセットを構成する

  1. インストールIDを見つけます。

    1. ブラウザから、管理ノードのREST APIUIにログインします。

    2. ストレージMVIPに移動してログインします。このアクションにより、証明書は次のステップで受け入れられます。

    3. 管理ノードでインベントリサービスRESTAPIUIを開きます。

       https：// [管理ノードIP] / Inventory / 1 /

    4. [承認]をクリックして、以下を完了します。

      1. クラスターのユーザー名とパスワードを入力します。

      2. クライアントIDをmnode-clientとして入力します。

      3. [承認]をクリックしてセッションを開始します。

    5. REST API UIから、[ GET / installations ]をクリックします。

    6. [試してみる]をクリックします。

    7. [実行]をクリックします。

    8. コード200の応答本文から、後の手順で使用するためにインストール用のidをコピーして保存します。

      インストールには、インストールまたはアップグレード中に作成された基本資産構成があります。

  2. （NetApp HCIのみ）vSphereでコンピュートノードのハードウェアタグを見つけます。

    1. vSphere WebClientナビゲーターでホストを選択します。

    2. [監視]タブをクリックし、[ハードウェアの状態]をクリックします。

    3. ノードBIOSの製造元とモデル番号が一覧表示されます。後のステップで使用するために、 tagの値をコピーして保存します。

  3. NetApp HCIモニタリング（NetApp HCIインストールのみ）およびハイブリッドクラウドコントロール（すべてのインストール用）用のvCenterコントローラーアセットを管理ノードの既知のアセットに追加します。

    1. 管理ノードのIPアドレスに続けて/mnode入力して、管理ノードのmnodeサービスAPIUIにアクセスします。

       https：// [管理ノードIP] / mnode

    2. [認証]または任意のロックアイコンをクリックして、以下を完了します。

      1. クラスターのユーザー名とパスワードを入力します。

      2. クライアントIDをmnode-clientとして入力します。

      3. [承認]をクリックしてセッションを開始します。

      4. 窓を閉めて。

    3. POST / assets / {asset_id} / controllersをクリックして、コントローラーのサブアセットを追加します。

    4. [試してみる]をクリックします。

    5. クリップボードにコピーした親ベースアセットIDをasset_idフィールドに入力します。

    6. タイプvCenterおよびvCenter資格情報を使用して必要なペイロード値を入力します。

    7. [実行]をクリックします。

（NetApp HCIのみ）計算ノード資産を構成します

  1. （NetApp HCIの場合のみ）計算ノード資産を管理ノードの既知の資産に追加します。

    1. POST / assets / {asset_id} / compute-nodesをクリックして、計算ノード資産の資格情報を持つ計算ノードサブ資産を追加します。

    2. [試してみる]をクリックします。

    3. クリップボードにコピーした親ベースアセットIDをasset_idフィールドに入力します。

    4. ペイロードに、[モデル]タブで定義されている必要なペイロード値を入力します。 typeとしてESXi Hostを入力し、hardware_tagの前の手順で保存したハードウェアタグを入力しhardware_tag 。

    5. [実行]をクリックします。
