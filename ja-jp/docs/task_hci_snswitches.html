SN2010、SN2100、およびSN2700スイッチを交換してください

2020年12月1日寄稿者 このページのPDFをダウンロード

ネットアップが提供するベストプラクティスと手順に従うことで、障害のあるSN2000シリーズスイッチを無停止で交換できます。

必要なもの

  • Puttyがラップトップにインストールされていること、および出力をキャプチャしていることを確認してください。出力セッションをキャプチャするようにPuttyを構成する方法については、このビデオを参照してください。

  • 交換の前後に、NetApp ConfigAdvisorを実行していることを確認してください。これは、メンテナンスを開始する前に他の問題を特定するのに役立ちます。 Config Advisorをダウンロードしてインストールし、 ここからクイックスタートガイドにアクセスします（ログインが必要です） 。

  • 電源ケーブル、基本的な手工具、およびラベルを入手します。

  • 2〜4時間のメンテナンスウィンドウを計画していることを確認してください。

  • 以下のスイッチポートをよく理解してください。

図1：SN2010スイッチのフェースプレートとポート
図2：SN2100スイッチのフェースプレートとポート
図3：SN2010およびSN2100スイッチの背面
図4：SN2700スイッチの前面と背面
このタスクについて

この手順の手順は、以下の順序で実行する必要があります。これは、ダウンタイムを最小限に抑え、スイッチを交換する前に交換用スイッチを事前に構成するためです。

ガイダンスが必要な場合は、ネットアップサポートにお問い合わせください。

手順の手順の概要は次のとおりです。
故障したスイッチを交換する準備をする
構成ファイルを作成します
障害のあるスイッチを取り外し、交換品を取り付けます
スイッチのオペレーティングシステムのバージョンを確認します
交換用スイッチを構成します
交換を完了する

故障したスイッチを交換する準備をする

障害のあるスイッチを交換する前に、次の手順を実行してください。

ステップ

  1. 交換用スイッチが故障したスイッチと同じモデルであることを確認します。

  2. 障害のあるスイッチに接続されているすべてのケーブルにラベルを付けます。

  3. スイッチ構成ファイルが保存されている外部ファイルサーバーを特定します。

  4. 次の情報を取得していることを確認してください。

    1. 初期構成に使用されるインターフェース：RJ-45ポートまたはシリアルターミナルインターフェース。

    2. スイッチアクセスに必要な資格情報：障害のないスイッチと障害のあるスイッチの管理ポートのIPアドレス。

    3. 管理アクセス用のパスワード。

構成ファイルを作成します

作成した構成ファイルを使用して、スイッチを構成できます。次のいずれかのオプションから選択して、スイッチの構成ファイルを作成します。

オプション ステップ

障害のあるスイッチからバックアップ構成ファイルを作成します

  1. 次の例に示すように、SSHを使用してリモートでスイッチに接続します。

     ssh admin @ <switch_IP_address

  2. 次の例に示すように、設定モードに入ります。

    スイッチ>有効
スイッチ＃端末の構成

  3. 次の例に示すように、使用可能な構成ファイルを見つけます。

    スイッチ（構成）＃
switch（config）＃設定ファイルを表示

  4. アクティブなBIN構成ファイルを外部サーバーに保存します。

     switch（config）＃configuration upload my-filename scp：// myusername @ my-server / path / to / my / <file>

別のスイッチからファイルを変更して、バックアップ構成ファイルを作成します

  1. 次の例に示すように、SSHを使用してリモートでスイッチに接続します。

     ssh admin @ <switch_IP_address

  2. 次の例に示すように、設定モードに入ります。

    スイッチ>有効
スイッチ＃端末の構成

  3. 次の例に示すように、テキストベースの構成ファイルをスイッチから外部サーバーにアップロードします。

    スイッチ（構成）＃
switch（config）＃設定テキストファイルmy-filename upload scp：// root @ my-server / root / tmp / my-filename

  4. 障害のあるスイッチに一致するように、テキストファイルの次のフィールドを変更します。

     ##ネットワークインターフェース構成
##
インターフェイスなしmgmt0dhcp
   インターフェイスmgmt0IPアドレスXX.XXX.XX.XXX/ 22

##
##その他のIP構成
##
   ホスト名oldhostname

障害のあるスイッチを取り外し、交換品を取り付けます

障害のあるスイッチを取り外し、交換品を取り付ける手順を実行します。

ステップ

  1. 障害のあるスイッチの電源ケーブルを見つけます。

  2. スイッチの再起動後、電源ケーブルにラベルを付けてプラグを抜きます。

  3. 障害のあるスイッチからすべてのケーブルにラベルを付けてプラグを抜き、スイッチの交換中の損傷を防ぐためにケーブルを固定します。

  4. スイッチをラックから取り外します。

  5. 交換用スイッチをラックに取り付けます。

  6. 電源ケーブルと管理ポートケーブルを接続します。

    AC電源を投入すると、スイッチの電源が自動的にオンになります。電源ボタンはありません。システムステータスLEDが緑色に変わるまでに最大5分かかる場合があります。

  7. RJ-45管理ポートまたはシリアルターミナルインターフェイスを使用してスイッチに接続します。

スイッチのオペレーティングシステムのバージョンを確認します

スイッチのOSソフトウェアバージョンを確認します。障害のあるスイッチと正常なスイッチのバージョンが一致している必要があります。

ステップ

  1. SSHを使用してリモートでスイッチに接続します。

  2. 設定モードに入ります。

  3. 実行show versionコマンドを使用します。次の例を参照してください。

     SFPS-HCI-SW02-A（config）#show version
製品名：オニキス
製品リリース：3.7.1134
ビルドID：＃1-dev
ビルド日：2019-01-24 13:38:57
ターゲットアーチ：x86_64
ターゲットハードウェア：x86_64
作成者：jenkins @ e4f385ab3f49
バージョンの概要：X86_64 3.7.1134 2019-01-24 13:38:57 x86_64

製品モデル：x86onie
ホストID：506B4B3238F8
システムシリアル番号：MT1812X24570
システムUUID：27fe4e7a-3277-11e8-8000-506b4b891c00

稼働時間：307d 3h 6m 33.344s
CPU負荷の平均：2.40 / 2.27 / 2.21
CPUの数：4
システムメモリ：3525MB使用済み/ 3840MB空き/合計7365MB
スワップ：使用済み0MB /空き0MB /合計0MB

  4. バージョンが一致しない場合は、OSをアップグレードする必要があります。詳細については、 Mellanoxソフトウェアアップグレードガイドを参照してください。

交換用スイッチを構成します

交換用スイッチを構成する手順を実行します。詳細については、 Mellanoxの構成管理を参照してください。

ステップ

  1. 該当するオプションから選択してください。

オプション ステップ

BIN構成ファイルから

  1. 次の例に示すように、BIN構成ファイルをフェッチします。

     switch（config）＃configuration fetch scp：// myusername @ my-server / path / to / my / <file>

  2. 次の例に示すように、前の手順でフェッチしたBIN構成ファイルをロードします。

     switch（config）＃configuration switch-to my-filename

  3. yesと入力して、再起動を確認します。

テキストファイルから

  1. スイッチを工場出荷時のデフォルトにリセットします。

     switch（config）＃出荷時設定へのリセット-基本

  2. テキストベースの構成ファイルを適用します。

     switch（config）＃設定テキストファイルmy-filename apply

  3. 次の例に示すように、テキストベースの構成ファイルをスイッチから外部サーバーにアップロードします。

    スイッチ（構成）＃
switch（config）＃設定テキストファイルmy-filename upload scp：// root @ my-server / root / tmp / my-filename
    テキストファイルを適用するときに再起動する必要はありません。

交換を完了する

手順を実行して、交換手順を完了します。

ステップ

  1. ラベルを使用してケーブルを挿入します。

  2. NetApp ConfigAdvisorを実行します。 ここからクイックスタートガイドにアクセスします（ログインが必要です） 。

  3. ストレージ環境を確認します。

  4. 障害のあるスイッチをNetAppに戻します。
